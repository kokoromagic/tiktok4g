Tổng quan giải pháp

Nếu bạn không muốn tốn tiền thuê VPS hàng tháng hoặc gặp trở ngại do mạng nhà bị chặn NAT/CGNAT không có IP Public để mở cổng (Port Forwarding), giải pháp kết hợp giữa Xray-Core và Cloudflare Tunnel (trycloudflare.com) sẽ giúp bạn biến máy tính Windows, Linux hoặc điện thoại Android (qua Termux) thành một máy chủ VLESS chạy ngay tại nhà hoàn toàn miễn phí.

Cơ chế hoạt động dựa trên việc một tiến trình ngầm tạo đường hầm an toàn kết nối thẳng tới Edge Node của Cloudflare và cấp cho bạn một Subdomain ngẫu nhiên dạng *.trycloudflare.com, bỏ qua mọi rào cản gác cổng của nhà mạng.

Bước 1: Khởi chạy VLESS Server lần đầu

Dự án mã nguồn mở này đã được viết tối ưu bằng Python, tự động tải phiên bản Xray và Cloudflared phù hợp với hệ điều hành của bạn mà không cần cài đặt thủ công.

Mở Terminal / Command Prompt lên và chạy tuần tự các lệnh sau:

# Tải source code từ kho lưu trữ về máy
git clone https://github.com/vincentng295/xray_vless_ws_server

# Di chuyển vào thư mục dự án
cd xray_vless_ws_server

# Cài đặt các thư viện Python cần thiết 
pip install -r requirements.txt

# Bật server lên (Lần sau chỉ cần chạy lệnh này)
python main.py

Kết quả: Sau khi chạy, hệ thống sẽ ghi cấu hình v2ray vào trong file frp_info.config. Bạn chỉ cần sao chép link nhập vào ứng dụng v2rayNG/Shadowrocket để sử dụng.

💡 Mẹo Android: Bạn có thể cài đặt ứng dụng Termux từ F-Droid, gõ lệnh pkg install python git -y rồi chạy các bước tương tự để biến chiếc điện thoại cũ thành một server VPN mini chạy 24/7 cực kỳ tiết kiệm điện.

Bước 2: Cố định kết nối bằng Cloudflare Workers

Nhược điểm của các bản thử nghiệm ở Bước 1 là mỗi lần bạn khởi động lại script, Cloudflare sẽ cấp một subdomain ngẫu nhiên mới khiến bạn phải cập nhật lại liên kết trên điện thoại rất phiền phức. Để cố định nó, ta sử dụng Cloudflare Workers làm cầu nối trung gian vĩnh viễn.

Cách thiết lập tự động hóa:

  1. Mở file _worker.js trong thư mục dự án ra, sửa dòng if (password !== "xxxxxxxx") bằng mật khẩu bí mật của bạn. Tiến hành sao chép toàn bộ nội dung file này, dán vào trang quản trị Cloudflare Workers của bạn rồi nhấn Deploy. Bạn sẽ nhận được một URL cố định (Ví dụ: my-vless.username.workers.dev).
  2. Mở tiếp file cấu hình môi trường ẩn .env ở thư mục dự án trên máy tính/điện thoại ra và điền thông tin Webhook lưu lại:
WEBHOOK_URL=https://my-vless.username.workers.dev/setapi?password=mat_khau_cua_ban

Nguyên lý đồng bộ: Giờ đây, mỗi lần bạn chạy lệnh python main.py tại nhà, mã script sẽ tự động gửi tín hiệu thông báo cho Cloudflare Worker biết đường hầm tạm thời hôm nay là gì để lưu trữ vào bộ nhớ đệm RAM. Khi kết nối từ xa trên điện thoại, bạn chỉ cần điền duy nhất tên miền cố định của Worker (my-vless.username.workers.dev). Lưu lượng WebSocket sẽ tự động được định tuyến ngầm chuyển tiếp mượt mà về server tại phòng của bạn một cách an toàn mà không cần thay đổi cấu hình trên Client lần nào nữa!

Bước 3: Cấu hình KV Storage chống mất cấu hình (Khuyên dùng)

Mặc định, biến GLOBAL_TARGET_HOST và GLOBAL_TARGET_PATH của Worker chỉ được lưu trên RAM tạm thời. Nếu Worker không có lượt truy cập trong thời gian ngắn hoặc Cloudflare khởi động lại hệ thống, RAM sẽ bị xóa sạch, khiến bạn gặp lỗi 503 Service Unavailable rất khó chịu.

Để khắc phục triệt để, bạn hãy gán bộ nhớ Workers KV nhằm lưu cấu hình vĩnh viễn theo các bước trực quan dưới đây:

  1. 1. Truy cập Workers KV: Tại trang Dashboard Cloudflare của bạn, điều hướng tới menu bên trái chọn Storage & databases → Click vào mục Workers KV.
  2. 2. Tạo không gian lưu trữ (Instance): Nhấp vào nút xanh Create Instance (hoặc Create Namespace) ở góc trên bên phải.
  3. 3. Đặt tên Namespace: Nhập tên bạn muốn đặt (Ví dụ: tuandeptrai_vless) rồi ấn nút Create để xác nhận lưu.
  4. 4. Vào phần Cài đặt của Worker: Quay lại giao diện quản lý đoạn code Worker của bạn (dự án private-server), chọn tab Settings ở thanh menu trên cùng.
  5. 5. Đi tới liên kết tài nguyên (Bindings): Di chuyển tiếp sang tab phụ tên là Bindings bên cạnh mục Cấu hình chung.
  6. 6. Thêm Binding mới: Nhấn vào nút xanh Add binding + và chọn dòng KV namespace ở bảng hiển thị ra.
  7. 7. Ánh xạ biến môi trường: Điền chính xác ô Variable name là KV_CONFIG (bắt buộc đúng chữ hoa để khớp với code trong Worker). Ở ô KV namespace, hãy bấm chọn tên phân vùng bạn vừa tạo ở Bước 3. Cuối cùng, ấn nút Add Binding (hoặc Save) để hoàn tất.

🎉 Hoàn thành: Giờ đây Worker của bạn đã được trang bị "ổ cứng" lưu trữ. Kể từ lần Python đồng bộ webhook tiếp theo, các thông số cấu hình sẽ tự động được ghi giữ lại mãi mãi, không bao giờ lo chết kết nối đột ngột nữa!

Nguyên lý hoạt động (Bẻ băng thông)

Giải thích đơn giản nhất: Khi bạn đăng ký gói TikTok, nhà mạng sẽ mở đường truyền "thả ga" cho riêng ứng dụng này. Máy chủ VLESS đóng vai trò như một "vỏ bọc ngụy trang". Khi bạn vào Facebook, YouTube, hay lướt Web, phần mềm VPN trên máy sẽ gom toàn bộ dữ liệu đó lại, "dán nhãn" giả vờ bên ngoài là lưu lượng xem TikTok rồi gửi đi. Nhà mạng kiểm tra thấy nhãn TikTok nên cho đi qua miễn phí. Khi gói dữ liệu đến được máy chủ VLESS, máy chủ sẽ bóc lớp vỏ ngụy trang này ra và giúp bạn kết nối tới trang web đích mong muốn.

Hướng dẫn đăng ký gói cước TikTok các nhà mạng

Để xài 4G Miễn phí thông qua server, bạn cần đăng ký gói cước TikTok tương ứng với nhà mạng đang sử dụng:

1. Viettel (Chọn gói theo nhu cầu):

  • Dưới 50GB/tháng: Soạn T50K gửi 191 (50k → có 50GB/tháng)
  • Dưới 100GB/tháng: Soạn T15KN gửi 191 (15k/tuần → có 25GB/tuần. Dùng cả tháng hết 60k có 100GB, tự động gia hạn theo tuần)
  • Trên 100GB/tháng: Soạn T5K gửi 191 (5k/ngày → có 15GB/ngày. Dùng cả tháng hết 150k có 450GB, tự động gia hạn theo ngày)
  • Gói không giới hạn: Soạn MXH100 gửi 191 (100k → Không giới hạn nền/tháng. Chỉ áp dụng cho thuê bao di động trả trước kích hoạt mới từ 01/01/2023)

2. VinaPhone (Thực hiện lần lượt các bước):

  • Soạn TK30 gửi 888 → 30k/không giới hạn nền/tháng
  • Nên để sim 0đ tránh bị trừ tiền hoặc truy cập MoMo: Vào mục Data 4G/5G → Thẻ data → Vinaphone → Chọn gói 1,5k/50MB/30 ngày → Mua ngay → Nạp.

3. MobiFone (Chọn 1 trong các gói):

  • Soạn DK TIK30 gửi 9199 → 50k/50GB/tháng
  • Soạn TT1 gửi 9199 → 3K/ngày (không giới hạn dung lượng nền)
  • Soạn DK TT80 gửi 9199 → 80k/không giới hạn nền/tháng

Android (Rooted) - Magic V2Ray Module

Magic V2Ray là một công cụ mạng nâng cao được thiết kế riêng cho điện thoại Android đã root. Bằng cách kết hợp các lõi proxy hàng đầu hiện nay, dự án tạo ra một kết nối mượt mà xuyên suốt toàn hệ thống và bao phủ mọi ứng dụng của bạn.

  • Tải xuống bản phát hành .zip mới nhất từ trang GitHub chính thức của dự án Magic V2Ray.
  • Flash file module thông qua trình quản lý Magisk hoặc KernelSU manager, sau đó tiến hành khởi động lại thiết bị.
  • Mở giao diện cấu hình Web UI cục bộ của ứng dụng bằng cách nhấn nút Action bên trong quản lý Magisk / KernelSU.
  • Điền Link đăng ký vào ô nhập và nhấn nút Process Connect Link để tải danh sách cấu hình.
